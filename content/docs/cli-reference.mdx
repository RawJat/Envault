# CLI Reference

Envault comes with a powerful CLI to manage your secrets directly from your terminal.

## Installation

You can run Envault directly using `npx` (recommended) or install it globally.

<Tabs items={['npx (Recommended)', 'Global Install']}>
  <Tab value="npx (Recommended)">
  ```bash
  npx @dinanathdash/envault login
  ```
  </Tab>
  <Tab value="Global Install">
  ```bash
  npm install -g @dinanathdash/envault
  ```
  </Tab>
</Tabs>

## Commands

### `login`

Authenticate your machine securely via Device Flow.

```bash
envault login
```

### `init`

Link your current folder to an Envault project. This creates a local configuration to track which project the secrets belong to.

```bash
envault init
```

### `deploy` (or `push`)

Push your local `.env` variables to the secure cloud vault.

```bash
envault deploy
# or
envault push
```

### `pull`

Fetch the latest secrets from the cloud and update your `.env` file.

```bash
envault pull
```

## Features

- <Lock className="inline w-4 h-4 mr-2" /> **End-to-End Encryption**: Secrets are encrypted before they leave your machine.
- <Key className="inline w-4 h-4 mr-2" /> **Device Flow Authentication**: Secure, browser-based login mechanism.
- <Shield className="inline w-4 h-4 mr-2" /> **Role-Based Access**: Enforces project permissions (Editor/Viewer) on deploy and pull operations.
- <Rocket className="inline w-4 h-4 mr-2" /> **Zero-Config**: Works with your existing `.env` files.
- <Zap className="inline w-4 h-4 mr-2" /> **Framework Agnostic**: Works with Next.js, Node.js, Python, Go, etc.
